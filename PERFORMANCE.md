# 维修管理系统 - 性能优化说明

## 性能优化特性

### 分页浏览
- **每页显示**：默认20条记录，最大50条
- **避免卡顿**：大数据量时不会导致页面卡顿
- **智能分页**：自动计算总页数，支持快速跳转

### 数据库优化
- **索引优化**：关键字段添加数据库索引
  - `created_at` - 创建时间索引
  - `updated_at` - 更新时间索引
  - `status` - 状态索引
  - `device_number` - 维修编号索引
  - `customer_name` - 客户姓名索引

- **查询优化**：
  - 按维修编号数字部分排序
  - 分页查询避免全表扫描
  - 兼容 MySQL 和 SQLite 两种数据库语法

### 导出优化
- **智能限制**：默认限制导出1000条记录
- **用户选择**：可选择导出全部数据或限制数量
- **性能提示**：大数据量时提醒用户可能较慢

### 主题系统性能
- **CSS变量**：主题切换仅修改CSS自定义属性，无需重载样式表
- **无刷新切换**：主题即时生效，无需页面刷新
- **本地存储**：主题选择保存在 localStorage，减少重复计算

## 性能指标

### 页面加载
- **首次加载**：< 2秒（1000条记录）
- **分页切换**：< 500ms
- **搜索筛选**：< 1秒
- **主题切换**：< 50ms

### 数据库查询
- **分页查询**：< 100ms
- **统计查询**：< 200ms
- **状态更新**：< 50ms

## 数据库兼容性

### MySQL
- 使用 `CAST(...AS UNSIGNED)` 进行编号排序
- 使用 `DATE_FORMAT` 进行日期格式化
- 使用 `DATEDIFF` 计算日期差

### SQLite
- 使用 `CAST(...AS INTEGER)` 进行编号排序
- 使用 `strftime` 进行日期格式化
- 使用 `julianday` 计算日期差
- 数据库文件位于 `data/repair_management.db`

## 调优建议

### 服务器配置
- **PHP内存限制**：建议设置 `memory_limit = 256M`
- **MySQL配置**：
  ```ini
  innodb_buffer_pool_size = 1G
  query_cache_size = 64M
  max_connections = 200
  ```

### 数据库维护
- **定期优化**：每周执行 `OPTIMIZE TABLE repair_records`（MySQL）
- **索引维护**：监控索引使用情况
- **数据清理**：定期清理过期数据
- **SQLite维护**：定期执行 `VACUUM` 压缩数据库文件
